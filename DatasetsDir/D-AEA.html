Description

Amazon.com-Employee Access Challenge (D-AEA): The goal of this competition is to create an algorithm that can predict approval/denial for an unseen employee,based on historical records about approval/denial of employeeâ€™s requests to access certainresources.9We use the winning code from Owen Zhang. It first converts the original categorical features to numerical features. It then builds six models from subsets of thefeatures as well as features obtained via postprocessing (e.g., aggregation). The final prediction is generated by an ensemble of predictions from individual models. The algorithmsit uses are GBM (generalized boosted regression modeling), random forest,extremely randomized trees, and glmnet (lasso and elasticnet regularized generalized linear models). Features used by this winning code are stored by unioning all features used by the models. We abbreviate this constructed dataset as D-AEA.

How to use: The column name of the Label is: label The features are contained in: Amazon_new_train.csv Amazon_new_test.csv The id of the test data is contained in: id.csv
